<task type="auto" tdd="true">
  <name>Task 1: 实现 release-summary.js 核心逻辑</name>
  <files>plugin/vibe/scripts/release-summary.js</files>
  <behavior>
    - `categorize(msg)`:
        - `feat(ui): add button` -> `feat`
        - `added new feature` -> `feat` (fuzzy)
        - `random message` -> `Internal Changes` (fallback)
    - `getTaskInfo(msg)`:
        - `done (task: T1)` -> 尝试读取 `.vibe/tasks/T1.json`，若成功提取 status/id 等信息，若失败则仅显示 ID。
    - `getGitLog(from, to)`: 
        - 自动检测 latest tag 到 HEAD。
        - 若无 tag，从首个 commit 开始。
  </behavior>
  <action>
    - 使用纯 Node.js 实现，不引入外部 npm 依赖。
    - 调用 `child_process.execSync` 执行 git 命令。
    - 实现分类引擎：
        - 正则匹配 `^(feat|fix|docs|test|refactor|chore)(?:\(.*\))?:`。
        - 模糊匹配关键字（例如 `feat`: [add, new, feat], `fix`: [fix, bug, patch] 等）。
        - 默认归类为 `Internal Changes`。
    - 实现任务解析：
        - 使用正则 `\(task:\s*(\w+)\)` 捕获 ID。
        - 尝试读取 `.vibe/tasks/<id>.json`，失败时不报错。
    - 支持 `--json` 和 `--from`/`--to` 参数。
    - 默认输出 `.vibe/RELEASE_DRAFT.md`。
  </action>
  <verify>
    <automated>node plugin/vibe/scripts/release-summary.test.js</automated>
  </verify>
  <done>所有单元测试通过，脚本逻辑完整。</done>
</task>
