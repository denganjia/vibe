       - 确定关联的脚本文件（约定：`skills/name/` 对应 `scripts/name.js`）。
    3. 动态注册 MCP 工具：
       - 工具名称：`vibe_skill_{name}` (例如 `vibe_skill_init`)。
       - 描述：使用 `SKILL.md` 中的 `description`。
       - 参数：定义一个通用的 `params: z.record(z.any()).describe("Skill-specific inputs as defined in SKILL.md")` 和可选的 `workspaceRoot: z.string()`。
    4. 实现通用的调用逻辑：
       - 根据工具名加载对应的脚本模块。
       - 将 `params` 传递给导出的函数。
       - 捕获输出并作为 MCP 响应内容返回。
  </action>
