<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| AI Client → MCP Server | 所有的工具输入（尤其是文件路径）必须经过校验以防止路径遍历。 |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-01 | Tampering | vibe_acquire_lock | mitigate | 使用 zod 校验 filePaths 为合法的相对路径，防止访问 .vibe 之外的敏感文件（虽然 MCP 默认在工作区运行，但防御性编程是必要的）。 |
| T-27-02 | Denial of Service | vibe_create_task | mitigate | 限制任务上下文和属性的字符串长度，防止恶意超大数据导致内存溢出。 |
</threat_model>
