<task type="auto">
  <name>Task 3: Implement Server-Side Signal Bus</name>
  <files>crates/vibe-core/src/ipc/server.rs</files>
  <action>
    Update `MasterServer` and `handle_connection`:
    - Add `waiters: Arc<Mutex<HashMap<String, Vec<mpsc::Sender<String>>>>>>` to `MasterServer`.
    - In `handle_connection`, when `Message::Wait(info)` is received:
        - Create an `mpsc::channel`.
        - Add the `tx` to the `waiters` list for `info.signal_name`.
        - **Block** the connection loop until a message is received from the channel (or timeout).
        - When a message (payload) is received, send `Message::SignalFired` back to the client.
    - When `Message::Signal(info)` is received:
        - Find all `tx` in the `waiters` list for `info.name`.
        - Send the `info.payload` as JSON to each `tx`.
        - Clear the list for that signal name.
        - Send `Message::Ack` back to the signaler.
  </action>
  <verify>
    <automated>cargo test -p vibe-core --lib ipc::server::tests</automated>
  </verify>
  <done>Server correctly handles Signal and Wait messages with asynchronous blocking.</done>
</task>
