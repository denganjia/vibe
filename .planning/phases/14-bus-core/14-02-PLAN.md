<task type="auto">
  <name>Task 2: 实现 vibe signal 命令</name>
  <files>apps/vibe-cli/src/main.rs</files>
  <action>
    - 在 `Commands` 枚举中添加 `Signal { name: String, payload: Option<String>, #[arg(long)] target: Option<String> }`。
    - 实现 `vibe signal` 逻辑：
        - 确定目标物理 ID：
            1. 如果提供 `--target`，使用该值。
            2. 如果环境变量 `VIBE_MASTER_ID` 存在，使用该值。
            3. 如果均不存在，且未指定 target，则视为广播（从 `StateStore` 获取所有活跃窗格）。
        - 格式化信号字符串：`\n[vibe-signal:<NAME>] <JSON_PAYLOAD>\n`。
        - 使用当前终端适配器的 `send_keys` (注入文本) 向目标窗格发送信号。
        - 注入时需小心，不要在 `send_keys` 内部额外添加会导致命令执行的换行符（除非必要）。
  </action>
  <verify>
    <automated>cargo build -p vibe-cli</automated>
  </verify>
  <done>`vibe signal` 能向指定窗格或广播注入格式化的信号文本。</done>
</task>
