<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user request -> Conductor protocol | Untrusted user intent becomes task contracts through model reasoning |
| task contract -> filesystem | `file_scope` controls which paths executors may touch |
| Agent contract -> subprocess | configured commands and env/context cross into external Agent CLIs |
| executor output -> reviewer | untrusted generated artifacts are assessed before completion |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-07 | Tampering | `task-contract.md` file scope | mitigate | Require repo-root-relative file scopes and explicit rejection of `..` path traversal. |
| T-20-08 | Information Disclosure | `agent-contract.md` subprocess context | mitigate | Define allowlisted environment variables and prohibit dumping secrets into artifacts. |
| T-20-09 | Tampering | `collaboration-protocol.md` model/script boundary | mitigate | State that `.vibe` task/review artifacts are synchronization source; scripts do deterministic work only. |
| T-20-10 | Repudiation | `review-protocol.md` findings | mitigate | Require structured findings with evidence and required fixes before completion. |
| T-20-11 | Denial of Service | concurrent executors | mitigate | Reference path-scoped ownership and lock expectations in task contract; implementation occurs in Phase 22. |
| T-20-12 | Spoofing | Agent definition | accept | Phase 20 documents Agent identity fields; authentication between local CLIs is outside this phase. |
</threat_model>
