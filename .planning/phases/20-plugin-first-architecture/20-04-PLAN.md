<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| legacy CLI -> plugin scripts | Old command responsibilities are re-owned by scripts, references, or compatibility paths |
| legacy env vars -> subprocess Agents | `VIBE_*` variables can carry context into external Agent processes |
| legacy pane state -> `.vibe` task state | old pane records must not become the primary progress source |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-19 | Tampering | migration matrix | mitigate | Require every old CLI command row to map to exactly `Migrate-to-script`, `Compatibility`, or `Remove`. |
| T-20-20 | Information Disclosure | `VIBE_PERSONA` and subprocess env | mitigate | Document allowlisted env passing and no arbitrary env dump to subprocess Agents. |
| T-20-21 | Tampering | legacy pane operations | mitigate | Classify pane lifecycle as Compatibility, not default execution path. |
| T-20-22 | Tampering | task file scopes | mitigate | Include no path traversal in migration checklist for script-owned file scopes. |
| T-20-23 | Denial of Service | concurrent legacy/runtime concepts | accept | Phase 20 classifies lock and bus concepts; implementation of lock expiry belongs to Phase 22. |
| T-20-24 | Repudiation | migration rationale | mitigate | Require rationale column and explicit compatibility reference file list. |
</threat_model>
