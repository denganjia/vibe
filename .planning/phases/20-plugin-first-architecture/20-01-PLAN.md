<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| plugin package -> AI terminal | Codex reads local plugin manifest, skills, and docs from `plugin/vibe/` |
| marketplace catalog -> plugin package | Codex local discovery metadata points from `.agents/plugins/marketplace.json` to `./plugin/vibe` |
| model -> scripts | Model instructions can request deterministic script actions, but scripts must not accept arbitrary policy text as shell |
| plugin template -> user workspace | Template files guide `.vibe` initialization and must not imply destructive overwrite behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-01 | Tampering | `plugin/vibe/scripts/README.md` | mitigate | Document that scripts only accept structured workspace/task inputs and do not decide policy or execute arbitrary model text. |
| T-20-02 | Tampering | `plugin/vibe/templates/.vibe/locks/README.md` | mitigate | Require repo-root path normalization and atomic create semantics for future lock files. |
| T-20-03 | Tampering | `.vibe` template docs | mitigate | Keep initialization documented as non-destructive template material; implementation belongs to Phase 21 and must preserve user files unless force is explicit. |
| T-20-04 | Information Disclosure | `logs/README.md` and run artifacts | mitigate | State `no secrets` in logs and require future scripts to avoid copying environment secrets into subprocess logs. |
| T-20-05 | Denial of Service | lock semantics | accept | Phase 20 documents lock contract only; runtime timeouts and stale lock cleanup are Phase 22 implementation concerns. |
| T-20-06 | Elevation of Privilege | plugin manifest | accept | Manifest only declares local plugin identity and skills path; no executable install hook is added in this plan. |
| T-20-25 | Spoofing | `.agents/plugins/marketplace.json` | mitigate | Pin the local source path to `./plugin/vibe` and keep it as local discovery metadata, not a remote marketplace or install hook. |
</threat_model>
