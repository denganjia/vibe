<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| command contract -> Conductor skill | User-facing command text triggers model workflow behavior |
| run-task command -> subprocess Agent | Task and Agent definitions cross into external model CLI execution |
| review-task command -> completion state | Reviewer output controls whether a task may be marked complete |
| release-summary command -> git history | Local repository metadata is summarized into a release draft |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-13 | Tampering | `run-task.md` | mitigate | Require structured task id input, task contract reads, subprocess default, and artifact capture instead of arbitrary shell text. |
| T-20-26 | Tampering | `plugin/vibe/commands/*.md` | mitigate | Require every command file to include `Command Contract Only` and state it does not claim executable Codex command binding. |
| T-20-14 | Information Disclosure | `run-task.md` subprocess capture | mitigate | Route secret rules through `agent-contract.md` and avoid dumping env secrets into run artifacts. |
| T-20-15 | Tampering | `review-task.md` completion gate | mitigate | Block completion on unresolved high severity findings. |
| T-20-16 | Repudiation | `status.md` | mitigate | Require status to read durable `.vibe` task/run/review/log/lock artifacts. |
| T-20-17 | Denial of Service | command-driven runtime | accept | Phase 20 creates command contracts only; runtime retry/timeouts are Phase 22 concerns. |
| T-20-18 | Elevation of Privilege | `release-summary.md` | mitigate | Explicitly local-only output and no network-required GitHub publishing in Phase 20/24 local draft path. |
</threat_model>
