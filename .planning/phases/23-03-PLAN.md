<task type="auto">
  <name>Task 2: Implement review-fix loop logic in review-task.js</name>
  <files>plugin/vibe/scripts/review-task.js</files>
  <action>
    - 更新 `plugin/vibe/scripts/review-task.js`：
        - 调用 Reviewer Agent 并获取其 findings。
        - 如果 findings 包含 `severity: high/critical`：
            - 检查任务的 `run_count` 是否达到 3。
            - 如果达到上限，设置状态为 `failed`。
            - 如果未达到，设置状态为 `fix-needed`，并将 `run_count` 加 1 (per D-03)。
        - 如果审查通过，设置状态为 `completed`。
  </action>
  <verify>
    <automated>
      # 模拟审查失败

      node plugin/vibe/scripts/review-task.js T1 --force-fail
      grep "fix-needed" .vibe/tasks/T1.json
    </automated>
  </verify>
  <done>review-task.js 正确处理审查结果并管理 fix-needed 状态与重试计数</done>
</task>
