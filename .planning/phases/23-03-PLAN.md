<task type="auto">
  <name>Task 2: Update plan.js for Review Processing</name>
  <files>plugin/vibe/scripts/plan.js</files>
  <action>
    - Add a `--process-review <task_id>` command to `plugin/vibe/scripts/plan.js` (per Feedback 2).
    - Logic (Implementation of D-03 and Feedback 2):
        1. Find the latest review file: `.vibe/reviews/{task_id}_run_{run_id}.json`.
        2. Read findings. If any have `severity === 'high'` or `'critical'`:
            - If `run_count < 3`:
                - Increment `run_count` in `task.json`.
                - Aggregate findings into task `context` or `constraints`.
                - Set status to `queued`.
            - If `run_count >= 3`:
                - Set status to `failed` and log an error.
        3. If findings pass (no high/critical):
            - Set status to `completed`.
    - Reference D-03: Conductor aggregates findings.
  </action>
  <verify>
    <automated>
      node plugin/vibe/scripts/task.js create '{"id": "t2", "status": "review-needed", "run_count": 0}' > /dev/null
      echo '[{"severity": "high", "message": "fix it"}]' > .vibe/reviews/t2_run_1.json
      node plugin/vibe/scripts/plan.js --process-review t2
      grep '"status": "queued"' .vibe/tasks/t2.json && grep "fix it" .vibe/tasks/t2.json
    </automated>
  </verify>
  <done>plan.js handles the fix loop orchestration, including aggregation and status reset.</done>
</task>
