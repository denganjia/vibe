  <name>Task 2: Implement dependency check in run-task.js</name>
  <files>plugin/vibe/scripts/run-task.js</files>
  <action>
    - 修改 `plugin/vibe/scripts/run-task.js` 的前置检查逻辑 (per D-02)：
        - 读取待执行任务的 `dependencies` 字段。
        - 遍历依赖 ID，检查对应的 `.vibe/tasks/<id>.json` 是否存在。
        - 如果任何依赖任务的 `status` 不是 `completed`：
            - 将当前任务状态设为 `blocked`。
            - 打印错误信息并退出（Exit 1）。
        - 如果所有依赖均已完成且状态为 `blocked`，将其重置为 `queued` 或 `running`（视锁情况而定）。
  </action>
  <verify>
    <automated>
      # 模拟依赖失败场景
      mkdir -p .vibe/tasks
      echo '{"id":"B", "status":"queued", "dependencies":["A"]}' > .vibe/tasks/B.json
      echo '{"id":"A", "status":"running"}' > .vibe/tasks/A.json
      node plugin/vibe/scripts/run-task.js B || [ $? -ne 0 ]
    </automated>
  </verify>
  <done>run-task.js 拒绝执行依赖未完成的任务并将其标为 blocked</done>
</task>
