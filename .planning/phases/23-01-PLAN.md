<task type="auto">
  <name>Task 2: Implement dependency check in run.js</name>
  <files>plugin/vibe/scripts/run.js</files>
  <action>
    - Add a `checkDependencies(task)` function to `plugin/vibe/scripts/run.js`.
    - Logic: Iterate through `task.dependencies`. For each ID, read `.vibe/tasks/{id}.json`.
    - A dependency is satisfied ONLY if its `status` is exactly `completed`.
    - If any dependency is not met:
        1. Log the missing/incomplete dependency IDs.
        2. Update the current task's status to `blocked` using the existing status update logic.
        3. Exit the process with code 1 to prevent execution.
    - Ensure pure Node.js implementation.
  </action>
  <verify>
    <automated>
      # Setup: Create dep task (queued) and dependent task
      node plugin/vibe/scripts/task.js create '{"id": "dep1", "status": "queued"}' > /dev/null
      node plugin/vibe/scripts/task.js create '{"id": "main", "dependencies": ["dep1"]}' > /dev/null
      # Run dependent task: should fail and mark as blocked
      node plugin/vibe/scripts/run.js main || grep '"status": "blocked"' .vibe/tasks/main.json
    </automated>
  </verify>
  <done>run.js enforces dependency order by blocking tasks with incomplete prerequisites.</done>
</task>
