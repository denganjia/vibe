  <name>Task 1: 初始化工作区根目录与 vibe-core 核心库</name>
  <files>Cargo.toml, crates/vibe-core/Cargo.toml, crates/vibe-core/src/lib.rs</files>
  <action>
    1. 创建 `crates/vibe-core/src` 目录。
    2. 将原 `src/` 下的 `adapter/`, `env.rs`, `error.rs`, `os/`, `state/` 迁移至 `crates/vibe-core/src/`。
    3. 创建 `crates/vibe-core/src/lib.rs`，按顺序暴露这些模块：
       ```rust
       pub mod error;
       pub mod env;
       pub mod adapter;
       pub mod os;
       pub mod state;
       ```
    4. 创建 `crates/vibe-core/Cargo.toml`，迁移除 `clap` 以外的所有依赖。
    5. 重写根目录 `Cargo.toml` 为工作区模式：
       ```toml
       [workspace]
       resolver = "2"
       members = ["apps/*", "crates/*"]

       [workspace.package]
       version = "0.1.0"
       edition = "2024"
       authors = ["Anjia"]

       [workspace.dependencies]
       tokio = { version = "1", features = ["full"] }
       rusqlite = { version = "0.32", features = ["bundled"] }
       serde = { version = "1.0", features = ["derive"] }
       serde_json = "1.0"
       thiserror = "2.0"
       anyhow = "1.0"
       dirs = "5.0"
       which = "6.0"
       clap = { version = "4.5", features = ["derive"] }
       vibe-core = { path = "crates/vibe-core" }
       ```
  </action>
  <verify>
    <automated>ls crates/vibe-core/src/lib.rs && cat Cargo.toml | grep "\[workspace\]"</automated>
  </verify>
  <done>vibe-core 基础结构就绪，核心逻辑迁移完成。</done>

       [package]
       name = "vibe-cli"
