# .vibe/reviews

## Owned By

Review files are owned by reviewer Agents and the Conductor workflow that decides whether a task can complete.

## File Naming

Use task-derived names such as `task-001-review-001.json` so findings remain attached to the reviewed task and run.

Reviews should include structured findings, severity, required fixes, and reviewer sign-off before completion.

## Safety Notes

A task should not be marked complete until required fixes are resolved or explicitly accepted by the Conductor.
