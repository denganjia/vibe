# SPEC Phase 1: Definition

## Goal
Define the contract and specifications for a new feature or component.

## Instructions
1. **Gather Requirements**: Identify the inputs, outputs, and behaviors.
2. **Draft Specification**: Create a structured SPEC file (e.g., `REQUIREMENTS.md` or a JSON schema).
3. **Verify Constraints**: Ensure the specification aligns with existing system architecture.
4. **Validation**: Use tools to validate the spec (e.g., linting, schema validation).
5. **Report**: Call `vibe report` to signal the spec is ready for review.

## Next Step
Transition to the SDD lifecycle to implement the specification.
